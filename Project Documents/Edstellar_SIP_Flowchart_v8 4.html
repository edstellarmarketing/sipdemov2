🔁 Platform Shifts to Recurring Usage Model from This Point
Once the first LNA/TNA is generated, the platform has an employee baseline. Every action from here — a training completed, an assessment cycle run, a new hire joining, a promotion, a strategy change, a Transformation Dial shift, a tool added — creates a delta against that baseline. The platform's value is in tracking change over time. This is the heartbeat of continuous workforce intelligence.

⚠️ 6 Modules From Product Document v18 — Not Represented in the SIP Flow

These modules are real product features but do not belong as steps in the skills intelligence process flow. Each has a reason.

📈 Section 22 — Freemium Conversion Engine GTM / Sales
This is a commercial acquisition process, not a SIP workflow step. It describes how Edstellar converts free-tier users to paying customers — upgrade triggers, locked feature previews, ROI calculators. This belongs in a GTM or Sales Funnel flowchart, not in the skills intelligence process flow experienced by a client already on the platform.
🌍 Section 26 — Localisation Specification Engineering / Infrastructure
This is an infrastructure and engineering configuration layer that sits beneath every step in the flow. RTL support, Works Council documentation, EEOC assessment review, and cultural calendar awareness are configuration concerns that affect how the platform is delivered in each market — not distinct steps that a user experiences in the workflow.
🛒 Section 24 — Assessment Content Marketplace Secondary Marketplace / Commercial Product
This is a secondary marketplace product that sits on top of the assessment infrastructure. Organisations buying and selling assessment content between each other is a commercial feature — useful, but not a step in the skills intelligence workflow. It enriches the question bank available to a client but does not constitute a distinct phase in the LNA → LNI → Action flow.
📊 Section 25 — Cross-Org Benchmarking Analytics Output / Premium Feature
This is an analytics output product, not a process step. Benchmarking data enhances the InsightBoard and strategic output layer (it informs the Predictive Layer's industry trend monitoring signal) but it does not constitute a distinct phase. It is a feature within Phase 10 Strategic Output, referenced there, rather than a separate step in the flow.
🔌 Section 21 — Public API & Developer Portal Technical Infrastructure
This is platform infrastructure. The API allows external tools (BI platforms, ATS systems, HRIS vendors) to consume SIP data — it is the mechanism by which other systems integrate with the flow, not a step within it. API endpoints are the plumbing that enables integrations. They should appear in a technical architecture diagram, not a process flow diagram.
🎮 Section 18 — Gamification & Engagement Mechanics UX Overlay / Motivation Layer
This is a motivation and engagement overlay that runs across all phases simultaneously — it is not a distinct step. Badges trigger when skills are validated (L2), streaks build as employees log learning (L3), team challenges run during assessment cycles (L2). Gamification is woven into the fabric of the flow rather than occupying its own position in it. It is represented in the Always-On Infrastructure strip at the top of the flowchart instead.
